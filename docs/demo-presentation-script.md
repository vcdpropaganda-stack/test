# Demo Presentation Script

## Objetivo

Guiar uma apresentacao interna da VLservice com narrativa clara, sem depender de improviso.

Tempo recomendado:

- 12 a 18 minutos

## Bloco 1. Abertura

Mensagem:

- a plataforma organiza a jornada inteira de servicos locais
- cliente publica a necessidade
- prestadores respondem
- o contato evolui ate contratacao, atendimento e avaliacao

Tela sugerida:

- `/`

## Bloco 2. Cliente publica e acompanha pedido

Conta:

- `vlservice+cliente2@gmail.com`

Telas:

- `/dashboard/client/pedidos`
- `/pedidos/estrutura-digital-home-office-campinas`

Historia:

- Lucas ja publicou um pedido para organizar atendimento digital e home office
- a plataforma consolidou as propostas recebidas
- o cliente consegue comparar valor, prazo e mensagem

Pontos para falar:

- mural de pedidos
- comparacao de propostas
- decisao com base em contexto, nao so em preco

## Bloco 3. Prestador recebe oportunidade

Conta:

- `vlservice+bruno@gmail.com`

Telas:

- `/dashboard/provider/pedidos`
- `/pedidos/faxina-apartamento-vila-mariana`

Historia:

- Bruno entra no painel e ve o mural priorizado
- os pedidos mais relevantes aparecem com criterio de cidade e categoria
- a conta tambem exibe saldo e regra de lances

Pontos para falar:

- feed priorizado
- proposta estruturada
- monetizacao simulada via creditos demo

## Bloco 4. Chat e contratacao

Conta:

- `vlservice+cliente3@gmail.com`

Telas:

- `/dashboard/mensagens`
- conversa com Carla Beauty

Historia:

- Fernanda ja contratou atendimento para evento corporativo
- existe conversa ativa com contexto real
- o cliente acompanha tudo dentro da plataforma

Pontos para falar:

- historico de mensagens
- fluxo de relacionamento
- continuidade entre proposta, booking e atendimento

## Bloco 5. Agendamentos

Conta:

- `vlservice+cliente3@gmail.com`

Telas:

- `/dashboard/client/agendamentos`
- booking pendente

Historia:

- um agendamento ainda esta pendente
- outro fluxo ja esta confirmado
- isso ajuda a mostrar ciclo operacional completo

Pontos para falar:

- estado pendente
- estado confirmado
- historico do cliente

## Bloco 6. Historico concluido e reputacao

Conta:

- `vlservice+cliente1@gmail.com`

Telas:

- `/dashboard/client/agendamentos`
- conversa fechada com Ana Clean

Historia:

- Mariana ja concluiu um atendimento
- ha review do cliente para o prestador
- ha review do prestador para o cliente

Pontos para falar:

- confianca
- reputacao bilateral
- retencao dentro da plataforma

## Bloco 7. Operacao e admin

Conta:

- `vlservice+admin@gmail.com`

Telas:

- dashboard/admin
- visoes de suporte/moderacao que existirem na build atual

Historia:

- o admin acompanha a base
- consegue validar pedidos, perfis e operacao

Pontos para falar:

- controle interno
- operacao assistida
- visibilidade do marketplace

## Fechamento

Mensagem final sugerida:

- hoje a demo mostra a jornada ponta a ponta
- cliente entra pelo problema
- prestador encontra oportunidade
- a conversa evolui ate a entrega
- a operacao ganha visibilidade e previsibilidade

## Fallback Rapido

Se alguma tela especifica falhar ao vivo:

- volte para `/dashboard/client/pedidos`
- mostre `/dashboard/provider/pedidos`
- abra `/dashboard/mensagens`

Esse trio sustenta quase toda a narrativa de valor da demo.
